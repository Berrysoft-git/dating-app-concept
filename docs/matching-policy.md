# 매칭 정책서

상태: 제품/운영 초안
작성일: 2026-05-21
기준 문서: `docs/prd.md`, `docs/mvp-scope.md`, `docs/member-tier-policy.md`, `docs/exposure-policy.md`
대상 서비스: 프리미엄 검증 기반 AI 소개팅 서비스

## 1. 목적

이 문서는 MVP에서 추천 후보군을 어떻게 만들고, 어떤 조건으로 제외하며, AI 추천 이유와 운영자 큐레이션을 어떤 범위에서 사용할지 정의한다.

핵심 원칙은 AI가 최종 판단자가 아니라 설명과 운영 보조 역할을 맡고, 후보군 생성은 회원 등급, 검증 상태, opt-in, 차단/신고, 계정 상태 같은 명시 정책을 먼저 적용하는 것이다.

## 2. 기본 원칙

- MVP는 무한 탐색형이 아니라 제한 추천형으로 운영한다.
- 후보군 생성은 정책 필터를 먼저 적용하고, AI 설명은 그 뒤에 생성한다.
- 일반 회원은 opt-in하지 않으면 셀렉트/프리미엄 회원에게 추천 후보로 들어가지 않는다.
- 셀렉트/프리미엄 회원 목록은 일반 회원에게 직접 공개하지 않는다.
- 차단, 신고 처리 중, 계정 제한, 탈퇴 예정 상태는 추천에서 우선 제외한다.
- AI 추천에는 인증자료 원본, OCR 원문, 고유식별번호, 관리자 내부 메모를 넣지 않는다.
- 추천 이유는 가능성, 공통점, 조율 포인트 중심으로 작성한다.
- 운영자 큐레이션은 정책 필터를 우회할 수 없다.

## 3. 추천 후보군 유형

| 후보군 | 대상 | 설명 |
|---|---|---|
| general_pool | 일반 회원 간 추천 | 일반 회원과 추천 가능 상태의 회원을 대상으로 한 기본 후보군 |
| select_pool | 셀렉트/프리미엄 회원 간 추천 | 검증 유효 상태의 셀렉트/프리미엄 회원 후보군 |
| select_to_general_opt_in_pool | 셀렉트 회원에게 추천 가능한 일반 회원 | opt-in한 일반/인기 일반 회원만 포함 |
| admin_curated_pool | 운영자 큐레이션 후보군 | 정책 필터를 통과한 후보 중 운영자가 수동 조정 |

MVP에서는 사용자가 직접 전체 회원을 검색하는 탐색형 후보군을 만들지 않는다.

## 4. 추천 가능 기본 조건

추천 후보가 되려면 다음 조건을 모두 만족해야 한다.

- 계정 상태가 활성이다.
- 필수 프로필이 완료되어 있다.
- 필수 사진 정책을 통과했다.
- 필수 설문을 완료했다.
- 탈퇴 예정 상태가 아니다.
- 중대한 미해결 신고 또는 제재가 없다.
- 추천 노출에 필요한 동의가 있다.
- 상대와 차단 관계가 아니다.

셀렉트/프리미엄 후보군에서는 추가로 검증 상태가 승인이고 유효기간이 지나지 않아야 한다.

## 5. 필수 제외 조건

아래 조건은 추천 점수, 인기 일반 신호, 운영자 큐레이션보다 먼저 적용한다.

| 제외 조건 | 처리 |
|---|---|
| 계정 제한 | 모든 후보군에서 제외 |
| 탈퇴 예정 또는 탈퇴 완료 | 모든 후보군에서 제외 |
| 프로필 미완성 | 모든 후보군에서 제외 |
| 필수 설문 미완료 | 모든 후보군에서 제외 |
| 부적합 사진만 보유 | 추천 카드 노출 제외 |
| 상호 또는 단방향 차단 | 양쪽 신규 추천 후보군에서 제외 |
| 중대 신고 처리 중 | 운영 검토 종료 전까지 제외 |
| 셀렉트 검증 만료 | 셀렉트 후보군에서 제외 |
| 일반 회원 셀렉트 opt-in 미동의 | 셀렉트 회원에게 추천되는 후보군에서 제외 |
| 사용자가 숨김 처리한 후보 | 정해진 기간 동안 재노출 제외 |

## 6. 후보군 생성 순서

MVP 후보군 생성 순서는 다음과 같다.

1. 요청 사용자 계정 상태 확인
2. 요청 사용자의 추천 받을 권한 확인
3. 추천 후보군 유형 결정
4. 필수 제외 조건 적용
5. 회원 등급, 검증 상태, opt-in 상태 적용
6. 차단/신고/제재 상태 적용
7. 선호 조건과 기본 호환 조건 적용
8. 인기 일반 신호 또는 품질 신호 적용
9. 운영자 큐레이션 후보 조정
10. 추천 카드 생성
11. AI 추천 이유 생성
12. 추천 카드 전달

AI 추천 이유는 후보군을 만들기 전에 후보를 허용하거나 금지하는 판단자로 쓰지 않는다.

## 7. 선호 조건과 호환 조건

MVP에서 사용할 수 있는 조건:

- 나이대
- 지역 단위
- 관계 목표
- 생활 패턴
- 대화 스타일
- 가치관 설문 요약
- 프로필 공개 범위
- 추천 노출 범위
- 차단/신고/숨김 이력

MVP에서 직접 점수화하거나 우열 판단에 쓰지 않을 조건:

- 소득/자산 세부 금액
- 회사명이나 학교명 자체의 서열화
- 외모 평가 점수
- 민감한 사생활 추정 정보
- 관리자 내부 메모
- 인증자료 원본 내용

## 8. 인기 일반 신호 사용 범위

인기 일반 신호는 추천 우선순위 보조 신호로만 사용한다.

허용:

- 프로필 완성도
- 설문 완료도
- 추천 카드 응답률
- 신고/차단 비율이 낮음
- 운영자 품질 피드백
- 부정행위 이력 없음

금지:

- 공개 순위 표시
- "상위 일반 회원" 같은 우열 문구
- 셀렉트/프리미엄 검증과 같은 배지 표시
- 인기 일반 신호만으로 셀렉트/프리미엄 권한 부여

인기 일반 회원도 셀렉트 추천 opt-in을 하지 않으면 셀렉트/프리미엄 회원에게 추천 후보로 들어갈 수 없다.

## 9. AI 추천 이유

AI 추천 이유는 추천 카드에 붙는 설명이다.

### 입력 가능 정보

- 익명 후보 ID
- 연령대
- 지역 단위
- 공개 가능한 관심사
- 성향/가치관 요약 feature
- 관계 목표 요약
- 대화 스타일 요약
- 추천 노출 동의 상태
- 검증 원본이 아닌 검증 상태와 공개 허용된 뱃지 정보
- 모델/프롬프트/feature 버전

### 입력 금지 정보

- 실명
- 전화번호, 이메일, 주소
- 인증자료 원본
- OCR 원문
- 주민등록번호 등 고유식별정보
- 소득/자산 증빙 원문
- 결제 정보
- 관리자 내부 메모 원문
- 신고/제재 내부 상세
- 사용자가 공개하지 않은 검증 세부 정보

### 출력 구조 초안

```json
{
  "reason": "공개 가능한 공통점과 잘 맞을 가능성 요약",
  "caution": "초반에 조율이 필요할 수 있는 지점",
  "confidence": "low | medium | high",
  "source_version": "feature/prompt/model version"
}
```

### 표현 기준

허용:

- 가능성이 있습니다
- 비슷한 경향이 있습니다
- 조율이 필요할 수 있습니다
- 대화 시작점이 될 수 있습니다

금지:

- 반드시 잘 맞습니다
- 더 우수한 상대입니다
- 조건이 더 좋습니다
- 소득이 높아서 안정적입니다
- 일반 회원보다 프리미엄 회원이 더 낫습니다

## 10. 추천 카드 액션

사용자 액션:

- 수락
- 거절
- 숨김
- 신고
- 차단
- 추천 이유 신고 또는 피드백

정책 기준:

- 같은 추천 카드에 대한 중복 액션은 한 번만 처리한다.
- 수락/거절/숨김 후 상태 전이는 되돌릴 수 있는지 MVP에서 별도 결정한다.
- 신고 또는 차단은 향후 후보군 생성에 즉시 반영한다.
- 숨김은 일정 기간 재노출을 막는다.
- 신고가 중대하면 운영 검토가 끝날 때까지 상대 노출을 제한한다.

## 11. 운영자 큐레이션

운영자 큐레이션은 초기 MVP에서 추천 품질을 보완하는 도구다.

허용 범위:

- 정책 필터를 통과한 후보의 우선순위 조정
- 추천 카드 발송 전 품질 검토
- 신고/제재 리스크가 낮은 후보 중심의 수동 추천
- AI 추천 이유의 부적절 표현 검토
- 제한된 테스트 그룹 대상 추천 조정

금지 범위:

- opt-in하지 않은 일반 회원을 셀렉트/프리미엄 후보군에 넣기
- 제한 또는 탈퇴 예정 계정을 추천하기
- 차단 관계를 무시하고 추천하기
- 검증 만료 회원을 셀렉트/프리미엄으로 추천하기
- 내부 메모나 민감정보를 사용자 추천 이유에 노출하기

운영자 큐레이션은 사유 코드와 처리자를 남긴다.

## 12. MVP 성공 지표와 연결

매칭 정책은 다음 지표로 검증한다.

| 지표 | 해석 |
|---|---|
| 추천 카드 수락률 | 후보군과 설명 품질 |
| 추천 이유 열람률 | 설명 가능한 매칭 가치 |
| 추천 이유 신고율 | 표현 정책 리스크 |
| 일반 회원 opt-in 비율 | 셀렉트 후보 참여 모델 수용도 |
| 셀렉트 회원 추천 수락률 | 프리미엄 풀 큐레이션 품질 |
| 신고/차단 비율 | 안전 리스크 |
| 숨김 비율 | 후보군 품질 또는 노출 피로 |
| 재방문율 | 제한 추천 방식의 지속성 |

## 13. 미확정 쟁점

다음은 개발/운영 검토 후 확정한다.

- 이미 생성된 추천 카드가 opt-out 후 자동 회수되어야 하는지
- 수락/거절 액션의 되돌리기 허용 여부
- 인기 일반 신호의 최소 기준과 운영자 검토 방식
- AI 추천 이유 생성 실패 시 fallback 문구
- 추천 카드 일일 제공 수와 재노출 제한 기간
- 운영자 큐레이션과 자동 후보군 점수의 우선순위
