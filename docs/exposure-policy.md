# 노출 정책서

상태: 제품/운영 초안
작성일: 2026-05-21
기준 문서: `docs/member-tier-policy.md`, `docs/select-premium-verification-criteria.md`, `docs/privacy-verification-data-policy.md`
대상 서비스: 프리미엄 검증 기반 AI 소개팅 서비스

## 1. 목적

이 문서는 일반 회원, 인기 일반 회원, 셀렉트/프리미엄 회원이 서비스 안에서 어떤 범위로 노출되는지 정의한다.

핵심은 셀렉트/프리미엄 회원의 직접 공개 탐색을 막고, 일반 회원도 명시적으로 opt-in한 경우에만 셀렉트/프리미엄 회원에게 추천 후보로 들어가도록 하는 것이다.

## 2. 기본 원칙

- 일반 회원은 셀렉트/프리미엄 회원 목록을 직접 조회할 수 없다.
- 일반 회원은 명시적으로 opt-in한 경우에만 셀렉트/프리미엄 추천 후보군에 포함된다.
- 인기 일반 회원은 셀렉트/프리미엄 검증을 통과한 회원이 아니다.
- 셀렉트/프리미엄 회원도 전체 셀렉트/프리미엄 회원 목록을 무제한 탐색하지 않는다.
- 추천 카드는 공개 가능한 프로필 정보와 설명 가능한 추천 이유만 보여준다.
- 검증 뱃지, 검증 유형, 추천 이유는 우열 표현이 아니라 사실 확인 중심으로 표시한다.
- 차단, 신고 처리 중, 계정 제한, 탈퇴 예정 상태는 추천 노출보다 우선한다.

## 3. 노출 대상 구분

| 대상 | 정의 | 직접 조회 | 추천 카드 노출 |
|---|---|---|---|
| 일반 회원 | 기본 가입과 추천 준비를 완료한 회원 | 일반 탐색 기능은 MVP에서 제한 | 일반 추천 풀에 노출 가능 |
| 인기 일반 회원 | 활동 품질과 신뢰 지표가 좋은 일반 회원 | 별도 공개 목록 없음 | opt-in 시 셀렉트 후보군에서 우선 신호로 사용 가능 |
| 셀렉트/프리미엄 회원 | 강한 자격 검증을 통과한 회원 | 일반 회원에게 목록 비공개 | 셀렉트 풀 또는 opt-in 후보에게 제한 노출 |
| 제한 회원 | 신고/정책 위반 등으로 제한된 계정 | 비공개 | 추천 후보 제외 |
| 탈퇴 예정 회원 | 삭제 요청 후 보존 기간 중인 계정 | 비공개 | 추천 후보 제외 |

## 4. 일반 회원 노출

일반 회원은 다음 조건을 모두 충족해야 일반 추천 후보군에 들어갈 수 있다.

- 계정 상태가 활성이다.
- 필수 프로필이 완료되어 있다.
- 필수 사진 정책을 통과했다.
- 필수 설문을 완료했다.
- 미해결 중대 신고 또는 제재가 없다.
- 탈퇴 예정 상태가 아니다.

일반 회원은 셀렉트/프리미엄 회원 목록을 직접 볼 수 없다. 셀렉트/프리미엄 회원에게 추천 후보로 들어가려면 별도 opt-in이 필요하다.

## 5. 인기 일반 회원 노출

인기 일반 회원은 별도 등급 자랑 배지가 아니라 추천 품질 신호로 다룬다.

인기 일반 후보 신호:

- 프로필 완성도
- 사진 정책 통과 여부
- 설문 완료도
- 추천 카드 응답률
- 신고/차단 비율
- 운영자 품질 피드백
- 부정행위 이력 없음

표시 제한:

- "인기", "상위", "우수"처럼 사람의 우열로 읽힐 수 있는 문구는 MVP에서 공개 배지로 쓰지 않는다.
- 셀렉트/프리미엄 회원과 같은 검증 뱃지로 오해되게 표시하지 않는다.
- 내부 추천 우선순위 신호로 쓰되, 사용자에게 직접 점수나 순위를 보여주지 않는다.

## 6. 셀렉트/프리미엄 회원 노출

셀렉트/프리미엄 회원은 다음 후보군에 제한적으로 노출된다.

| 노출 대상 | 허용 여부 | 조건 |
|---|---|---|
| 다른 셀렉트/프리미엄 회원 | 허용 | 양쪽 계정 활성, 추천 조건 충족, 차단/신고 없음 |
| opt-in 일반 회원 | 허용 | 일반 회원이 셀렉트 추천 opt-in 상태 |
| opt-in 인기 일반 회원 | 허용 | opt-in 상태이며 품질 신호가 안전 기준 통과 |
| opt-out 일반 회원 | 금지 | 명시 동의 없음 |
| 제한/탈퇴 예정 회원 | 금지 | 안전/계정 상태 우선 |

일반 회원이 셀렉트/프리미엄 회원 전체 목록을 훑는 기능은 MVP에 포함하지 않는다.

## 7. 일반 회원의 셀렉트 추천 opt-in

### opt-in 전 고지

opt-in 화면은 다음을 명확히 고지해야 한다.

- 셀렉트/프리미엄 회원 목록을 직접 볼 수 있는 기능이 아니다.
- opt-in하면 셀렉트/프리미엄 회원에게 추천 후보로 들어갈 수 있다.
- 추천 여부는 보장되지 않는다.
- opt-in은 언제든 철회할 수 있다.
- 철회 후에는 신규 셀렉트/프리미엄 추천 후보군에 포함되지 않는다.
- 이미 생성된 추천 카드가 있을 수 있으며, 철회 이후 처리 기준은 서비스 정책에 따른다.

### opt-in 상태

| 상태 | 의미 | 추천 후보 포함 |
|---|---|---|
| not_opted_in | 동의하지 않음 | 셀렉트 후보군 제외 |
| opted_in | 동의함 | 셀렉트 후보군 포함 가능 |
| opted_out | 철회함 | 신규 셀렉트 후보군 제외 |
| suspended | 안전/정책 사유로 일시 중지 | 제외 |

### opt-out 처리

사용자가 철회하면 다음 원칙을 적용한다.

- 신규 셀렉트/프리미엄 추천 후보군 생성에서 즉시 제외한다.
- 아직 전달되지 않은 추천 카드는 발송하지 않는다.
- 이미 상대에게 전달된 추천 카드는 회수 가능 여부를 별도 정책으로 정한다.
- 철회 행위와 시각은 감사 또는 운영 로그로 남긴다.

MVP에서는 이미 전달된 추천 카드의 자동 회수보다 신규 노출 차단을 우선 구현한다. 자동 회수 정책은 개발 구현 가능성 검토 후 확정한다.

## 8. 추천 카드 노출 정보

추천 카드에는 다음 정보를 표시할 수 있다.

- 닉네임 또는 표시명
- 나이대
- 지역 단위
- 공개 설정된 사진
- 공개 가능한 자기소개 요약
- 공개 설정된 관심사, 성향 요약
- 검증 완료 여부
- 사용자가 공개 허용한 검증 유형
- AI 추천 이유 요약

추천 카드에는 다음 정보를 표시하지 않는다.

- 실명
- 전화번호, 이메일, 주소
- 인증자료 원본 또는 OCR 원문
- 소득/자산 세부 금액
- 회사 내부 정보
- 관리자 내부 메모
- 신고/제재 내부 기록
- 사용자가 공개하지 않은 검증 세부 자료

## 9. 뱃지와 검증 표시

허용 표현:

- 검증 완료
- 재직 확인
- 전문 자격 확인
- 사업/임원 정보 확인
- 공개 활동 확인

금지 표현:

- 상위 회원
- 고급 회원
- 일반 회원보다 우수
- 고소득 우수 회원
- 엘리트 보장
- 선택받은 회원

검증 뱃지는 객관적 확인 상태만 나타내며, 추천 우선순위나 사람의 가치를 의미하지 않는다.

## 10. 차단, 신고, 제한 상태

다음 조건은 모든 노출 정책보다 우선한다.

| 조건 | 처리 |
|---|---|
| A가 B를 차단 | A에게 B를 노출하지 않고, B에게 A를 신규 노출하지 않는다. |
| 상호 차단 | 양쪽 추천 후보군에서 제외한다. |
| 중대 신고 처리 중 | 운영 검토가 끝날 때까지 신규 추천 후보군에서 제외한다. |
| 계정 제한 | 모든 추천 후보군에서 제외한다. |
| 탈퇴 예정 | 신규 추천 후보군에서 제외한다. |
| 검증 만료 | 셀렉트/프리미엄 노출 권한을 회수한다. |

## 11. AI 추천 이유 노출 기준

AI 추천 이유는 다음 기준을 따른다.

- 가능성, 공통점, 조율 포인트 중심으로 쓴다.
- 확정적 궁합 판단을 피한다.
- 소득, 자산, 직업, 학력, 외모를 우열 표현에 쓰지 않는다.
- 인증자료 원본에서 나온 민감 원문을 사용하지 않는다.
- 사용자가 신고하거나 숨길 수 있어야 한다.

추천 이유 예시:

- "생활 리듬과 대화 선호가 비슷해 초반 대화가 편할 가능성이 있습니다."
- "서로 일 중심의 일정이 많아 만남 빈도 조율이 필요할 수 있습니다."

금지 예시:

- "상대가 더 높은 조건이므로 좋은 기회입니다."
- "소득이 높아 안정적인 상대입니다."
- "일반 회원보다 프리미엄 회원이 더 우수합니다."

## 12. 운영자 큐레이션 노출

운영자는 초기 MVP에서 추천 품질을 보완하기 위해 큐레이션에 개입할 수 있다.

운영자 큐레이션 원칙:

- 정책상 제외 대상은 운영자가 임의로 추천 후보에 넣을 수 없다.
- opt-in하지 않은 일반 회원을 셀렉트/프리미엄 후보군에 넣을 수 없다.
- 제한, 탈퇴 예정, 중대 신고 처리 중인 회원은 추천할 수 없다.
- 수동 추천 사유는 내부 근거와 사용자 표시 문구를 분리한다.
- 수동 추천 행위는 감사 로그 또는 운영 로그로 남긴다.

## 13. 개발/구현 담당 리뷰 요청

제품/운영 관점에서 `docs/exposure-policy.md`를 작성했다. 개발/구현 관점에서 아래를 이어서 검토해줘.

1. 일반 회원 opt-in과 셀렉트/프리미엄 목록 비공개를 API/DB에서 강제하는 방식
2. opt-out 후 이미 생성된 추천 카드의 회수 또는 신규 노출 차단 처리
3. 차단, 신고 처리 중, 계정 제한, 검증 만료가 후보군 생성보다 우선 적용되는 구조
4. 인기 일반 신호를 공개 배지가 아니라 내부 추천 신호로만 쓰는 모델
5. 검증 뱃지와 검증 유형 공개 범위를 사용자 설정으로 제어하는 방식
6. 운영자 수동 큐레이션 로그와 사용자 표시 추천 이유를 분리하는 방식
