# 셀렉트/프리미엄 검증 기준표

상태: 제품/운영 초안
기준 문서: `docs/member-tier-policy.md`, `docs/privacy-verification-data-policy.md`, `docs/prd.md`, `docs/mvp-scope.md`
대상 서비스: 프리미엄 검증 기반 AI 소개팅 서비스

## 1. 목적

이 문서는 셀렉트/프리미엄 회원이 되기 위한 검증 유형, 제출 가능 자료, 승인/반려 기준, 재검증 기준을 정의한다.
셀렉트/프리미엄 검증은 사람의 우열을 판정하는 절차가 아니라, 서비스가 약속한 “검증 가능한 신뢰 신호”를 운영자가 일관되게 확인하기 위한 절차다.

## 2. 검증 원칙

- 셀렉트/프리미엄 회원은 단순 프로필 인증이 아니라 강한 자격 검증을 통과한 회원이다.
- 검증 기준은 직업, 소득/자산, 전문 자격, 학력/경력, 공개 영향력 등 객관적 확인 가능성이 있는 항목으로 제한한다.
- 모든 검증 유형은 승인, 반려, 재제출 요청, 만료, 재검증 상태를 가진다.
- 인증자료 원본은 별도 보안 저장소에 제한 기간만 보관하고, DB에는 원본 파일이나 고유식별번호 원문을 저장하지 않는다.
- 검증 뱃지는 우열 표현이 아니라 “확인된 신뢰 신호” 표시로만 사용한다.
- 등급과 검증 결과는 AI 추천에서 직접적인 우열 표현으로 사용하지 않는다.
- 운영자는 자유 판단만으로 승인하지 않고, 증빙 유형과 사유 코드를 남긴다.

## 3. 공통 신청 조건

| 항목 | 기준 | 미충족 시 처리 |
|---|---|---|
| 성인 여부 | 서비스 이용 가능 연령 충족 | 신청 불가 또는 반려 |
| 계정 상태 | 활성 상태 | 제한/탈퇴 예정 계정은 신청 불가 |
| 프로필 완성 | 필수 프로필, 사진, 설문 완료 | 재제출 요청 또는 신청 보류 |
| 신고/제재 | 중대한 미해결 신고 또는 제재 없음 | 신청 보류 또는 반려 |
| 자료 동의 | 수집 목적, 보관 기간, 열람 주체, 파기 기준 동의 | 신청 불가 |
| 원본 제출 | 조작 가능성이 낮은 원본 또는 발급처 확인 자료 | 재제출 요청 |

## 4. 검증 유형별 기준표

| 검증 유형 | 승인 기준 초안 | 제출 가능 자료 예시 | 반려/재제출 사유 |
|---|---|---|---|
| 주요 기업/기관 재직 | 현재 재직 중이며 회사/기관 식별이 가능하다. | 재직증명서, 사내 이메일 확인, 명함, 직원 포털 일부 가림 캡처 | 퇴사 상태, 회사 식별 불가, 과도한 가림, 조작 의심 |
| 고소득 | 운영 기준 이상의 최근 소득을 확인할 수 있다. | 소득금액증명, 원천징수영수증, 급여명세 일부 가림본 | 기간 불명확, 금액 확인 불가, 식별번호 노출, 조작 의심 |
| 고자산 | 운영 기준 이상의 금융/부동산/사업 자산을 확인할 수 있다. | 잔고증명, 평가서, 등기/보유 증빙 일부 가림본 | 본인 소유 불명확, 시점 불명확, 과도한 민감정보 노출 |
| 전문 자격 | 유효한 전문 자격 또는 면허가 있다. | 자격증, 면허증, 협회/기관 조회 결과 | 만료, 취소, 조회 불가, 이름/소속 불일치 |
| 학력/경력 | 고신뢰 학력 또는 경력을 확인할 수 있다. | 졸업증명, 재학/수료 증명, 경력증명 | 발급처 불명확, 본인 여부 불명확, 허위 가능성 |
| 창업/임원/사업자 | 실제 사업 운영 또는 임원 지위를 확인할 수 있다. | 사업자등록, 법인등기, 재직/임원 증빙, 언론/공시 자료 | 휴폐업, 지위 불명확, 단순 명의 대여 의심 |
| 공개 영향력 | 지속적인 공개 활동과 영향력을 확인할 수 있다. | 공개 SNS/채널, 언론 기사, 플랫폼 인증, 팔로워/조회 지표 | 구매/조작 의심, 일시적 바이럴, 본인 확인 불가 |
| 초대/운영 추천 | 내부 기준상 신뢰 가능한 후보로 운영자가 추천했다. | 기존 회원 추천, 운영자 인터뷰, 외부 평판 확인 | 증빙 부족, 이해관계 미고지, 안전 리스크 |

제출 가능 자료는 예시이며, 실제 서비스에서는 국가별 법률, 발급처, 개인정보 처리 기준에 맞춰 조정한다.

## 5. 등급 승인 방식

### 단일 유형 승인

다음 중 하나의 유형에서 강한 검증이 완료되면 셀렉트/프리미엄 후보로 승인할 수 있다.

- 주요 기업/기관 재직
- 고소득
- 고자산
- 전문 자격
- 창업/임원/사업자
- 공개 영향력

### 복합 유형 승인

단일 항목은 약하지만 복수의 검증 신호가 결합되면 승인할 수 있다.

예시:

- 안정적인 경력 + 전문 자격
- 학력/경력 + 공개 영향력
- 창업/사업자 + 매출 또는 외부 평판
- 인기 일반 활동 품질 + 운영자 인터뷰 + 외부 신뢰 신호

복합 승인 시 운영자는 어떤 신호 조합으로 승인했는지 사유 코드를 남긴다.

## 6. 승인/반려/재제출 기준

| 결정 | 조건 | 사용자 표시 |
|---|---|---|
| 승인 | 기준 충족, 자료 신뢰 가능, 계정 안전 리스크 없음 | 검증 완료와 유효기간 |
| 재제출 요청 | 자료 일부 부족, 가림 과다, 기간 불명확, 추가 확인 필요 | 보완 항목과 제출 기한 |
| 반려 | 기준 미충족, 본인 여부 불명확, 조작 의심, 안전 리스크 | 사유 코드와 재신청 가능 여부 |
| 보류 | 신고/제재 검토 중, 외부 확인 대기 | 검토 중 상태 |
| 만료 | 유효기간 경과, 재검증 미완료 | 권한 만료와 재검증 안내 |

반려 사유는 사용자에게 전달 가능한 사유 코드와 내부 운영 메모를 분리한다. 내부 메모에는 차별적 표현, 민감 추정, 불필요한 사생활 정보를 남기지 않는다.

## 7. 유효기간과 재검증

| 검증 유형 | 유효기간 초안 | 재검증 방식 |
|---|---:|---|
| 주요 기업/기관 재직 | 6~12개월 | 현재 재직 여부 재확인 |
| 고소득 | 12개월 | 최근 소득 자료 재확인 |
| 고자산 | 6~12개월 | 기준 시점 자산 자료 재확인 |
| 전문 자격 | 자격 유효기간 또는 12개월 | 자격 상태 조회 |
| 학력/경력 | 장기 유효, 필요 시 재확인 | 발급처 또는 외부 조회 |
| 창업/임원/사업자 | 6~12개월 | 사업 상태/지위 재확인 |
| 공개 영향력 | 6개월 | 공개 채널 상태와 조작 리스크 확인 |
| 초대/운영 추천 | 6개월 | 활동 품질과 안전 지표 재검토 |

유효기간은 법무/운영 검토 전 초안이다. 만료 전에는 사용자에게 재검증 안내를 제공하고, 만료 후에는 셀렉트/프리미엄 추천 풀 권한을 회수한다.

## 8. 자료 처리 기준

| 단계 | 처리 |
|---|---|
| 업로드 | 보안 저장소에 임시 보관하고, 악성 파일 검사와 메타데이터 제거를 수행한다. |
| 열람 | 심사자는 열람 사유를 입력한 뒤 필요한 범위만 확인한다. |
| 저장 | DB에는 자료 원본이 아니라 보안 저장소 객체 ID, 검증 유형, 상태, 만료 시각, 삭제 상태를 저장한다. |
| 승인/반려 후 | 개인정보 및 인증자료 처리 정책의 보관 기간에 따라 원본을 삭제 대기열에 넣는다. |
| 재검증 | 이전 원본을 재사용하지 않고 새 자료를 제출받는다. |
| 감사 | 열람, 승인, 반려, 재제출 요청, 권한 변경을 감사 로그로 남긴다. |

## 9. 금지 기준

다음 경우에는 승인하지 않는다.

- 본인 소유 또는 본인 상태임을 확인할 수 없는 자료
- 조작, 합성, 대여, 구매 정황이 있는 자료
- 제3자의 개인정보가 과도하게 포함된 자료
- 고유식별번호, 계좌 전체 번호, 카드 전체 번호 등 불필요한 원문이 그대로 포함된 자료
- 혐오, 성적 착취, 사기, 폭력, 불법 영업 등 중대한 안전 리스크가 확인된 계정
- 검증 기준을 충족하더라도 서비스 안전 정책을 위반한 계정

## 10. 뱃지와 사용자 표시

| 표시 항목 | 사용자 공개 여부 | 비고 |
|---|---|---|
| 셀렉트/프리미엄 검증 여부 | 공개 가능 | 상세 자료는 공개하지 않는다. |
| 검증 유형 | 제한 공개 | 사용자가 공개 범위를 설정할 수 있어야 한다. |
| 세부 금액/기관 자료 | 비공개 | 소득/자산/자료 원문은 공개하지 않는다. |
| 유효기간 | 사용자 본인과 운영자에게 표시 | 만료 안내용 |
| 반려 사유 | 사용자 본인에게 제한 표시 | 사유 코드 중심 |
| 내부 심사 메모 | 비공개 | 운영/감사용 |

뱃지 문구는 “상위”, “고급”, “우수” 같은 우열 표현보다 “검증 완료”, “전문 자격 확인”, “재직 확인”처럼 사실 확인 중심으로 쓴다.

## 11. 운영자 체크리스트

심사자는 승인 전 다음 항목을 확인한다.

1. 제출 자료가 신청 검증 유형과 맞는가
2. 본인 여부가 확인되는가
3. 자료 발급 시점과 유효기간이 적절한가
4. 불필요한 민감정보가 노출되어 있지 않은가
5. 조작 또는 대여 정황이 없는가
6. 계정의 신고/제재 리스크가 없는가
7. 승인/반려 사유 코드가 기록되었는가
8. 원본 삭제 예정 시각과 감사 로그가 남았는가

## 12. MVP에서 확정할 항목과 이후 결정

MVP에서 확정:

- 검증 유형 분류
- 승인/재제출/반려/보류/만료 상태
- 원본 장기 보관 금지
- 심사자 열람 사유와 감사 로그
- 검증 뱃지의 우열 표현 금지

이후 결정:

- 고소득/고자산의 구체 금액 기준
- 주요 기업/기관 범위
- 공개 영향력의 최소 지표
- 초대/운영 추천의 남용 방지 기준
- 자동 외부 검증 연동 범위
- 법무 검토 후 실제 보관 기간

6. 검증 뱃지와 AI 추천 feature가 차별적 우열 표현으로 이어지지 않게 하는 데이터 경계
