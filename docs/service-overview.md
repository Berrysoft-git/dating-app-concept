# 서비스 개요서

상태: 제품/운영 초안  
대상 서비스: 프리미엄 검증 기반 AI 소개팅 서비스  
저장소: `Berrysoft-git/dating-app-concept`

## 1. 서비스 한 줄 정의

검증된 회원 풀과 AI 성향 분석을 기반으로, 사용자가 직접 끝없이 탐색하는 대신 신뢰 가능한 상대를 큐레이션받는 소개팅 서비스.

## 2. 서비스 목적

기존 소개팅 앱은 사진, 나이, 지역, 직업, 짧은 자기소개처럼 얕은 정보에 의존한다. 사용자는 많은 프로필을 직접 넘겨야 하고, 실제로 왜 잘 맞는지 알기 어렵다.

이 서비스는 다음 문제를 해결한다.

- 신뢰하기 어려운 프로필
- 너무 많은 탐색 피로
- 조건 중심 매칭의 얕은 판단
- 프리미엄 회원의 노출 부담
- 일반 회원과 검증 회원 간의 비대칭 기대치
- 매칭 이유를 설명하지 못하는 추천 시스템

## 3. 핵심 가치 제안

### 신뢰

프리미엄/셀렉트 회원은 단순 본인 인증이 아니라 직업, 소득, 자산, 전문 자격, 공개 영향력 등 더 강한 기준으로 검증한다.

### 큐레이션

사용자는 전체 회원 목록을 무한 탐색하지 않고, 운영자 큐레이션과 AI 추천을 통해 제한된 고품질 후보를 받는다.

### 설명 가능한 매칭

AI는 단순 점수보다 “왜 이 사람이 잘 맞을 수 있는지”, “초반에 어떤 차이를 조율해야 하는지”를 설명한다.

### 프리미엄 풀 보호

일반 회원은 셀렉트/프리미엄 회원 목록을 직접 조회하지 못한다. 일반 회원은 본인이 opt-in한 경우에만 셀렉트 회원에게 추천 후보로 들어갈 수 있다.

### 운영 품질

초기 MVP는 완전 자동 매칭보다 운영자 심사, 큐레이션, 신고 대응, 피드백 수집을 통해 품질을 통제한다.

## 4. 주요 사용자 유형

| 사용자 유형 | 설명 | 핵심 니즈 |
|---|---|---|
| 일반 회원 | 기본 가입 회원 | 신뢰 가능한 소개 기회, 부담 없는 프로필 작성 |
| 인기 일반 회원 | 프로필 품질, 응답률, 신고 이력, 피드백이 좋은 일반 회원 | 더 좋은 추천 기회 |
| 셀렉트/프리미엄 회원 | 강한 자격 검증을 통과한 회원 | 노출 통제, 고품질 큐레이션, 안전한 만남 |
| 운영자/심사자 | 검증, 반려, 신고, 큐레이션을 담당 | 일관된 정책, 감사 로그, 안전한 관리 도구 |

## 5. 회원 구조 요약

### 일반 회원

- 기본 프로필을 작성한다.
- 성향/가치관 설문에 응답한다.
- 일반 회원 또는 추천 가능한 후보군으로 매칭될 수 있다.
- 셀렉트 회원에게 추천되려면 별도 opt-in이 필요하다.
- 셀렉트 회원 목록은 직접 볼 수 없다.

### 인기 일반 회원

- 일반 회원 중 신뢰도와 활동 품질이 높은 회원이다.
- 추천 우선순위가 높아질 수 있다.
- 셀렉트 회원에게 추천되는 경우에도 본인의 opt-in이 필요하다.

### 셀렉트/프리미엄 회원

- 직업, 소득, 자산, 전문 자격, 학력, 영향력 등 강한 기준으로 검증한다.
- 셀렉트 회원끼리 별도 큐레이션을 받을 수 있다.
- 선택적으로 검증된 일반/인기 일반 회원 추천을 받을 수 있다.
- 자기 프로필의 노출 범위를 더 세밀하게 통제할 수 있어야 한다.

13. 피드백 수집과 추천 품질 개선

## 7. MVP 목표

초기 MVP의 목표는 모든 기능을 완성하는 것이 아니라, “검증된 사람을 안전하게 소개받는 경험”이 실제 수요가 있는지 검증하는 것이다.

MVP에서 검증할 가설:

- 사용자는 무한 탐색보다 제한된 고품질 추천을 선호한다.
- 셀렉트/프리미엄 회원은 직접 노출보다 통제된 큐레이션을 선호한다.
- 일반 회원은 명확한 opt-in 조건이 있으면 셀렉트 회원 추천 후보가 되는 것을 받아들일 수 있다.
- AI 추천 이유는 수락률과 신뢰감을 높일 수 있다.
- 초기에는 완전 자동 매칭보다 운영자 큐레이션이 품질 관리에 유리하다.

## 10. 성공 기준 초안

| 지표 | 의미 |
|---|---|
| 프로필 완성률 | 가입자가 실제 추천 가능한 상태까지 도달하는지 |
| 셀렉트 검증 신청률 | 강한 검증 모델에 수요가 있는지 |
| 검증 승인율/반려율 | 기준이 지나치게 빡빡하거나 느슨하지 않은지 |
| 일반 회원 opt-in 비율 | 일반 회원이 셀렉트 추천 후보 참여를 받아들이는지 |
| 추천 카드 수락률 | 큐레이션 품질이 충분한지 |
| AI 추천 이유 열람률 | 설명 가능한 매칭에 가치가 있는지 |
| 신고/차단 비율 | 안전성과 품질 리스크 |
| 재방문율 | 제한 추천 방식이 지속 사용을 만드는지 |

## 11. 운영 원칙

- 프리미엄 검증 기준은 명확해야 하지만 차별적으로 표현하면 안 된다.
- 소득, 직업, 자산, 학력은 우열 표현이 아니라 검증 가능한 신뢰 신호로 다룬다.
- 일반 회원의 자존감을 해치지 않도록 “프리미엄에게 선택받는 구조”처럼 보이지 않게 설계한다.
- 셀렉트 회원 보호와 일반 회원 기회 제공 사이의 균형을 유지한다.
- 인증자료 원본은 장기 보관하지 않는다.
- 공개 저장소에는 실제 개인정보, 인증자료, API 키, DB 접속 문자열을 포함하지 않는다.

## 12. 기술 방향 요약

- 사용자 앱은 `React + Vite + TypeScript + Capacitor` 단일 코드베이스를 우선한다.
- 표준 HTML DOM을 유지해 텍스트 선택, 스크래핑, 외부 주석 도구와 호환되게 한다.
- iOS/Android 네이티브 기능은 Capacitor 공식 플러그인을 통해 호출한다.
- 민감정보와 인증자료는 별도 보안 저장소와 서버 API 뒤에서 처리한다.
- AI 추천은 최종 판단자가 아니라 추천 이유 설명과 운영자 보조 역할로 시작한다.
